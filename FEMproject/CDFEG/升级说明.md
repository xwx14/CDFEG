# CDFEG 相对 el 项目的 C++ 升级说明

> 生成日期：2026-06-15
> 对比对象：
> - **新项目**：`FEMproject/CDFEG/`（核心库）+ `FEMproject/sample/El2D/`（二维弹性示例）
> - **旧项目**：`E:\mfelProject\RegTestFold\el`（FEPG 生成的 C 求解器）

---

## 一、本质定性

| 维度 | 旧项目 `el`（前身） | 新项目 `CDFEG`（升级版） |
|---|---|---|
| **语言/范式** | C（FEPG/CFEPG 自动生成） | 现代面向对象 C++14 |
| **代码组织** | 过程式 + 全局变量驱动 | 三层继承 + 虚函数多态 + 命名空间 `CDFEG` |
| **求解器** | SuperLU 开源库（源码内联进项目 `solv/`，40+ 个 C 文件随项目编译） | Eigen `SimplicialLDLT`（header-only 依赖） |
| **复用机制** | 每个算例独立生成一套源码 | 共享 `CDFEG.dll`，算例仅写派生类 |

旧 `el` 是 FEPG 生成的「一次性」求解器；新 `CDFEG` 是**可复用、可扩展的有限元框架**。两者求解同一物理问题——二维平面应变线弹性静力学（位移法 + 应力恢复）。

---

## 二、架构层改进（SOLID + DRY）

### 2.1 三层继承体系取代全局变量耦合

旧项目核心数据 `dofa/ida/unoda/fa/matrixa/elema` 全是 `mfel.h` 里的**全局变量**，模块间靠全局状态隐式通信；新项目建立清晰的继承链：

```
FEMData（有限元空间：节点/单元/材料）
  └─ PhyFieldData（物理场：自由度/边界/总刚组装/求解）eProgram() / solve() / uPhy()
       └─ ElementBase（单元基类：run() / uEle()）
            └─ IsoEleBase（等参单元：形函数 / 雅各比）
```

- **单一职责（S）**：每层只管一件事——空间数据、场方程、单元计算。
- **开闭原则（O）**：新增单元类型只需继承 `IsoEleBase` 重写 `shapeFun/run/uEle`，无需改库代码。
- **依赖倒置（D）**：物理场持有 `ElementBase*` 抽象指针，通过 `_types` 字符串集合自动分发单元。

### 2.2 双场分离 → 单场统一（重大简化）

旧项目位移和应力是**两个独立组件**：

- `ela`（位移场，解 `K·U=F`）
- `elb`（应力场，独立方程组，`mass=vol` 最小二乘恢复）

两者靠全局 `unoda/unodb` 传递，并各需一套编号/装配/求解流程。

新项目在**同一个 `ElDispFieldData` 物理场**内完成：

- `run()` 算单刚 → 总刚 → 求解位移
- `uEle()` 直接在位移单元里由应变反求积分点应力 + 节点加权平滑 + vonMises 等效应力

**收益**：省掉一整个方程组的编号/装配/求解，DRY 原则体现充分。

## 三、单元与数值方法改进

### 3.1 等参单元的泛化

| 项目 | 形函数 | 雅各比矩阵 | 维度支持 |
|---|---|---|---|
| 旧 `a1eq4g2.c` | `switch` 硬编码 Q4，调用外部 `cshap.c` 的 `dshap` | **中心差分**（外部 cfepg 库 `dshap`/`dcoor`/`dcoef`） | 固化 2D |
| 新 `IsoEleBase` | 纯虚 `shapeFun()`，子类实现 | **中心差分**（内化进框架，`H=0.02`，支持 1 阶 + 2 阶 Hessian） | 1D/2D/3D 通用（`_iPow3s` 偏移编码表） |

> **澄清**：新旧项目**都用中心差分法求雅各比矩阵**——这是共同继承自 FEPG/cfepg 的做法，并非新项目首创。新项目的真正改进在于：① 把差分逻辑从外部 cfepg 库**内化进框架**（不再依赖外部 `cshap.c`）；② 用统一的偏移编码表 `_iPow3s` **泛化到 1/2/3 维**；③ 预计算并支持二阶（Hessian）导数。

`_refShapCoef` 预计算各积分点偏移位置的形函数值，把差分求导从运行时挪到初始化期，兼顾**泛化性与性能**。

> 注：旧项目 `eela.c` 中单元分发为 `switch(ityp)` 硬编码；新项目改为 `_types` set 匹配 + 虚函数分发，扩展性显著增强。

### 3.2 应力恢复算法升级

- **旧**：独立 `beq4g2` 单元 + 集中质量最小二乘，写入 `unodb`。
- **新**：`uEle()` 内一次完成——积分点应力 `s=D·e` → 按形函数 `weight` 加权外推到节点 → 全局加权平均。同时新增 **vonMises 等效应力**输出。

### 3.3 本构与刚度（保持物理等价）

新旧单刚公式等价，平面应变本构因子一致：

```
fact  = E / [(1+ν)(1-2ν)]
shear = 0.5 - ν           （即 G 经 fact 归一化）
stif  = eexx·eexx·(1-ν)·fact + eexx·eeyy·ν·fact + eeyy·eexx·ν·fact
      + eeyy·eeyy·(1-ν)·fact + eexy·eexy·shear·fact
```

差异仅在表达方式：旧用 1 基索引 + `kvord` 变量映射表，新用 0 基索引 + 直接循环。

## 四、工程基础设施改进

### 4.1 内存安全：裸指针 → STL + RAII

```
旧：double *a = calloc(maxa+1, ...);  int node[500];  #define maxnode 10000000
   // 固定大数组、无判空、手动 free 散落
新：std::vector<double> _data;  std::map<int,int> _colMap;
   // 自管理、按需增长、析构自动释放
```

彻底消除旧项目 `eela.c` 中 `calloc` 未判空、`free` 散落、固定上限 `maxtype[20]` 的隐患。

### 4.2 索引：1-based → 0-based

旧代码 `for(j=1; j<=nnode; ++j) node[j]=...` 全程 Fortran 风格 1 基索引；新代码符合 C++ 习惯 0 基索引，降低心智负担与越界风险。

### 4.3 求解器：精简依赖

- **旧**：将 LBNL 开源的 **SuperLU** 库源码整库内联（vendor）进项目 `solution/solv/`，含 **40+ 个 C 文件**（colamd、dgstrf、dgssv、panel_bmod、panel_dfs、sp_ienv …）随项目一同编译。SuperLU 本身并非手写，而是成熟开源稀疏直接求解器；问题在于**源码内联导致体积膨胀、版本固定、维护割裂**。
- **新**：`EquationSystem` 类封装自定义 CSR（`_numCol` / `_colId` / `_data` + `_colMap`）+ `convert2Eigen()`，求解改用 Eigen `SimplicialLDLT`，仅需 header-only 依赖。

**改进点**：以 header-only 的 Eigen 取代内联的 SuperLU 源码树，显著缩减项目体积、降低源码维护负担、并让线性代数工具（矩阵/向量/分解）统一在同一依赖下。

### 4.4 边界条件语义清晰化

| 项目 | 第一类边界（强制位移） | 第二类边界（力） |
|---|---|---|
| 旧 | `nodvar<0` 标记，在 `eela` 内手改右端项 `u -= K·u0` | `ubfa` 叠加 |
| 新 | `applyFirstBCs` 划行划列法，存 `_data0` / `_f0` 原始数据 | `applySecondBCs` 叠加右端项 |

新增 `calRightVals()`：用解得的位移反算右端项，可输出**支座反力**与做校验——旧项目无此能力。

### 4.5 前后处理：单一 GiD → 多格式

| 项目 | 网格输入 | 结果输出 |
|---|---|---|
| 旧 | 写死 GiD（`gidpre` / `gidmsh`） | 写死 GiD（`gidres`） |
| 新 | GiD + Abaqus INP（`inpReader`） | VTK/VTU（`vtkPost`）+ GiD 新版（`gidPrePost`）+ GiD 旧版兼容（`GidPrePost2`） |

网格来源与结果输出均解耦，不再绑死单一前后处理器。

## 五、工程化与入口

```cpp
// 旧：el.c
void main() { it=0; gidpre(); ... gidres(); }   // 无返回值、无参数、路径写死

// 新：main.cpp
int main(int argc, char* argv[]) {              // 命令行参数、可复用
    elData data;
    CDFEG::GidPrePost gidPrePost(&data);
    gidPrePost.setFilePath(path, project);      // 路径参数化
    gidPrePost.pre();
    data.caculate();
    gidPrePost.post2();
}
```

其它工程化要素：

- 所有源文件头附 **GPL-3.0 SPDX** 许可证声明
- **Doxygen** 风格注释（含 `@author` / `@date` / `@brief`）
- **DLL 导出宏** `CDFEG_API`，Windows 下经 `CDFEG_EXPORTS` 控制 `dllexport/dllimport`
- **CMake** 构建体系，核心库编译为 SHARED DLL，示例为独立可执行

## 六、设计原则视角的总评

| 原则 | 体现 |
|---|---|
| **KISS** | 沿用差分雅各比（与旧项目同为 FEPG 风格，`H=0.02`），但将其内化进框架、用偏移编码表统一 1/2/3 维，避免重写解析导数，以轻微截断误差换取**维度泛化 + 代码简洁** |
| **DRY** | 内联 SuperLU（40+ 文件）→ Eigen（header-only）；双场装配 → 单场；固定大数组 → 容器 |
| **YAGNI** | 框架只暴露虚函数接口，不做未来特性预埋，算例按需继承 |
| **OCP / SOLID** | 单元 / 物理场 / 数据三层均可继承扩展而不改库代码 |

## 七、工程取舍提示

新旧项目**均采用中心差分法**求雅各比矩阵（共同源自 FEPG/cfepg），因此差分本身并非新项目的取舍点。新项目的取舍在于：**继续沿用差分**（而非改写解析形函数导数），换取维度泛化与代码简洁；代价是对网格畸变敏感度高时精度略低于解析法。若后续做高精度分析，可在 `IsoEleBase` 派生层补充解析形函数导数。

## 八、新旧执行流程对照

**旧项目（双场两阶段）：**

```
main() → gidpre() 网格/材料/边界
       → starta() 位移场编号 + 稀疏符号分解
       → eela()    装配位移总刚（a1eq4g2 体 + a2ll2 面力）
       → solva()   SuperLU 求解 K·U=F
       → uela()    位移写回 unoda
       → eelb()    应力恢复（独立方程组，读 unoda 位移）
       → gidres()  输出位移 + 应力
```

**新项目（单场统一）：**

```
main() → gidPrePost.pre()           读网格/材料/边界
       → initMatrix()               方程编号 + CSR 骨架初始化
       → eProgram_el()              遍历单元 run() 装配总刚 + 应用边界
       → solve()                    Eigen SimplicialLDLT 求解
       → uPhy()                     位移写回 + uEle() 应力恢复 + 节点平滑 + vonMises
       → equSys.calRightVals()      反算支座反力
       → gidPrePost.post2()         输出位移 + 应力
```

## 九、结论

从 `el` 到 `CDFEG`，是一次从「FEPG 生成的过程式 C 单体程序」到「面向对象、可复用 DLL 框架 + 极简算例」的系统性重构。核心收益：

- **依赖精简**：SuperLU（40+ 文件）→ Eigen（header-only）
- **复用提升**：每个算例从「生成一整套源码」变为「写几个派生类 + main」
- **扩展友好**：单元 / 物理场 / 数据三层多态注册，符合开闭原则
- **内存安全**：裸指针 + 全局变量 → STL 容器 + RAII
- **设计取向**：单场统一、单元多态分发充分体现 KISS / DRY / YAGNI

可作为固体力学有限元位移法 + 应力平滑恢复的现代 C++ 参考实现。
