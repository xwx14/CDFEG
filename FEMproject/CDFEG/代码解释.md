# CDFEG 核心库代码解释

## 一、FemData、PhyFieldData、ElementBase 三个类及其层次关系

### 1. 总体层次结构

```
FEMData（有限元空间数据：节点/单元/材料管理）
  └── PhyFieldData（物理场数据：自由度/边界条件/总刚组装/求解）
        └── ElementBase（单元基类：run()/uEle() 计算单刚与后处理）或IsoEleBase（等参单元基类：形函数/雅各比矩阵/高斯积分）
```

这是一个**三层组合**的架构：

- **FEMData** 包含 `std::vector<PhyFieldData*> _phyDatas`（组合多个物理场）
- **PhyFieldData** 包含 `std::vector<ElementBase*> _eleSubs`（组合多个单元子程序）和 `EquationSystem _equSys`（方程系统）
- **ElementBase** 为虚基类，在其子类中实现单元计算程序（计算刚度矩阵、阻尼矩阵、质量矩阵等）以及单元后处理程序（计算各种物理量）。
- **IsoEleBase** 继承自 ElementBase，是等参元的基类。

### 2. FEMData — 有限元空间数据（`FemData.h/.cpp`）

**职责**：管理有限元模型的"网格数据"——节点坐标、单元连接关系、材料参数。

**关键成员**：
- `_nPts` — 节点总数
- `_nodes` — 节点坐标（一维 vector，行优先：`[x0,y0,z0, x1,y1,z1, ...]`）
- `_eleNodes` — 各单元的节点编号（扁平存储）
- `_elePt` — 各单元在 `_eleNodes` 中的起始位置索引（CSR 风格，长度为单元数+1，最后一个值为总长度）
- `_eleTypes` — 各单元的 VTK 类型
- `_dim` — 空间维度（1/2/3）
- `_nodeIdMap` — 外部节点 ID → 内部连续编号的映射
- `_eleIdMap` — 外部单元 ID → 内部连续编号的映射
- `_edgeIdMap` — edge 单元 ID → 所属单元 ID 的映射
- `_mateParams` / `_mateNames` — 材料参数列表（每个材料为一个 map<string,double>）
- `_eleMateIds` — 每个单元的材料号
- `_phyDatas` — 该模型关联的所有物理场
- `_dt` / `_tMax` — 时间步长和最大时间（用于瞬态分析）

**关键方法**：
- `addNode(id, x, y, z)` — 添加节点，外部 ID 转为内部 0~n-1 编号
- `addNodeEnd()` — 节点添加完毕，自动调用 `setNPts`
- `addEle(id, nodeIds, eleType)` — 添加单元，根据 eleType 字符串匹配到对应单元子程序
- `addEdge(id, nodeIds, eleType)` — 添加边/面单元
- `addMate(map, name)` — 添加材料
- `setEleMateId` / `setEleMateByName` — 设置单元材料
- `getElemMatParams(eleID, ele)` — 获取单元的材料参数
- `caculate()` / `main()` — **虚函数**，供派生类实现具体计算流程

### 3. PhyFieldData — 物理场数据（`PhyFieldData.h/.cpp`）

**职责**：管理一个物理场的自由度编号、边界条件、总刚组装和求解。

**关键成员**：
- `_dof` — 每个节点的自由度数（如弹性力学为 2 或 3）
- `_kVar` — 总自由度数 = 节点数 × _dof
- `_ida` — 自由度→方程号的映射（-1 表示该自由度不存在，≥0 为方程编号）
- `_neq` — 独立方程总数
- `_nodeBC1s` / `_nodeBC2s` — 第一类/第二类边界条件 `<dofId, val>`
- `_equSys` — `EquationSystem` 对象，存储稀疏矩阵并求解
- `_eleSubs` — 该物理场注册的所有单元子程序
- `_nodeRes` / `_elemRes` — 节点/单元计算结果
- `_dispNames` / `_nodeResNames` / `_eleResNames` — 结果名称标识

**关键方法**：
- `setNPts(n)` — 设置节点数，初始化 `_ida` 和 `_f`

- `addEleSub(eleSub)` — 注册单元子程序

- `setFirstBoundry(nodeId, val, iDof)` — 设置第一类边界条件（位移约束）

- `setSecondBoundry(nodeId, val, iDof)` — 设置第二类边界条件（力边界）

- `initMatrix()` — **核心方法**：
  1. 遍历所有单元，根据单元包含的节点自由度构建方程编号（`_ida`）
  
  2. 确定每个方程行的非零元列号集合（通过单元自由度关联）
  
  3. 调用 `_equSys.init(mht)` 初始化 CSR 稀疏矩阵结构
  
- `eProgram()` — **总刚组装**虚函数，默认调用eProgram_el()，在**子类中可重写**。
  
- `eProgram_el()` — **总刚组装**：
  1. 遍历每个单元子程序，对每个单元调用 `eleSub->run(r, coef, matParams)` 计算单刚
  2. 利用 `_ida` 构建单元自由度→方程号的映射（lm 数组）
  3. 调用 `_equSys.adda(estifn, lm)` 将单刚累加到总刚
  4. 应用第一类/第二类边界条件
  
- `solve()` — 委托 `_equSys.solve()` 求解线性方程组

- `uPhy()` — **后处理**虚函数，在**子类中可重写**，默认行为如下：
  1. 将求解结果 `_equSys._rhs` 按自由度映射回节点
  2. 遍历每个单元调用 `eleSub->uEle(r, coef, matParams)` 计算单元导出量（应力、应变等）

### 4. ElementBase — 单元基类（`ElementBase.h/.cpp`）

**职责**：所有单元类型的基类，定义了单元子程序的接口。

**关键成员**：
- `_nNode` — 每个单元的节点个数
- `_types` — 该单元类型匹配的字符串集合（如 `{"T3", "Tri3"}`）
- `_vtkCellType` — 对应的 VTK 单元类型
- `_phyData` / `_femData` — 所属物理场和有限元空间
- `_eleIds` — 属于该单元的 ID 列表
- `_dispNames` — 该单元参与的自由度名称
- `_nVar` — 单元未知量个数（= _nNode × 自由度）
- `_paramNames` — 单元子程序所需的材料参数名列表
- `_result` / `_results` — 实时计算结果/所有单元计算结果
- `_bSaveResult` — 是否保存每个单元的计算结果

**关键方法**（均为虚函数，由派生类实现）：

- `run(r, coef, matParams)` — 单元计算子程序：输入节点坐标、场系数、材料参数，输出 `EleSubResult`（含 `estif` 单刚、`emass` 质量矩阵、`eload` 载荷向量）
- `uEle(r, coef, matParams)` — 单元后处理：输入同上，输出 `uResult`（单元标量结果和节点向量结果）

**EleSubResult 结构**：
- `estif` — 单元刚度矩阵（一维 vector，行优先）
- `emass` — 单元质量矩阵
- `edamp` — 单元阻尼矩阵
- `eload` — 单元载荷向量
- `nodeIds` — 单元节点编号

### 5. IsoEleBase — 等参单元基类（`IsoEleBase.h/.cpp`）

**职责**：继承自 ElementBase，为等参单元提供形函数计算、有限差分法求雅各比矩阵和 Hessian 矩阵。

**关键成员**：
- `_nGaus` — 高斯积分点个数（默认等于节点数）
- `_nRefc` — 参考坐标维度
- `_refc` — 积分点参考坐标
- `_gaus` — 积分点权重
- `_refShapCoef` — `[iGaus][偏移][iNode]`，每个积分点的形函数值（含 H 偏移值，用于有限差分）
- `_dRefShapecoef` — 每个积分点的形函数雅各比矩阵
- `_ndord` — 导数最大阶数（1 或 2，支持 Hessian）

**关键方法**：
- **`shapeFun(refc)`** — **纯虚函数**，由具体单元类实现，计算参考坐标下的形函数值
- **`caculateShapeCoef(dim)`** — 在单元初始化时调用：
  1. 计算 `_iPow3 = 3^dim` 及 `_iPow3s`
  2. 对每个积分点，计算其形函数值及偏移 ±H 后的形函数值（用于有限差分）
  3. 调用 `dshap()` 计算形函数对参考坐标的导数
- **`dshap(iGaus, fx, dfdx, dord)`** — 用有限差分法计算形函数对参考坐标的雅各比矩阵（一阶）和 Hessian 矩阵（二阶）
  - 步长 H=0.02，中心差分公式
- **`dcoor(r, iGaus, fx, dfdx, dord)`** — 计算坐标变换的雅各比矩阵（全局坐标对参考坐标的导数）
- **`shapn(iGause, coor, crtr, cu)`** — 计算形函数对全局坐标的导数：
  - 利用链式法则：`dN/dx = dN/dr * dr/dx`
  - `crtr` 为雅各比矩阵的逆矩阵
- **`shapc()`** — 类似 shapn，但用于系数（非形函数）
- **`coordTransFun(r, refc)`** — 参考坐标→全局坐标变换
- **`coordTransFun(x, nVal, iGaus, i)`** — 利用预计算的形函数系数进行坐标/系数变换
- **`dCoef(x, iGaus, nVal, fx, dfdx, dord)`** — 对系数（如温度、位移）求有限差分导数

## 二、EquationSystem — 稀疏线性方程组（`EquationSystem.h/.cpp`）

**职责**：存储和管理稀疏线性方程组 `K·u = f`，支持自定义 CSR 格式，底层使用 Eigen 求解。

### CSR 自定义稀疏存储

使用标准的**压缩行存储（CSR）**格式：
- `_numCol` — 行指针数组，`_numCol[i]` 为第 i 行第一个非零元在 `_colId`/`_data` 中的索引，最后一项为总非零元数
- `_colId` — 列号数组，按行连续存储
- `_data` — 非零元数值数组
- `_colMap` — `vector<map<int,int>>`，每行中列号到 `_data` 索引的映射，用于 O(1) 快速定位添加元素

### 关键方法

- **`init(mht)`** — 从 `vector<set<int>>` 行结构初始化 CSR 矩阵
- **`adda(estifn, equIds)`** — 将单元刚度矩阵组装到总刚：遍历单元自由度映射，利用 `_colMap` 快速定位累加
- **`addFirstBC(equId, val)`** — 划行划列法处理第一类边界条件：
  1. 首次调用时备份原始 `_data` 到 `_data0`，`_f` 到 `_f0`
  2. 对角线设为 1，非对角元置 0
  3. 其他行中该列元素置 0，右端项减去 `a_ij * val`
  4. 支持多次调用、增量更新
- **`applyFirstBCs(bc1s, ida)`** — 批量应用第一类边界条件
- **`addSecondBC(equId, val)`** / **`applySecondBCs`** — 第二类边界条件直接叠加到右端项
- **`convert2Eigen()`** — 将 CSR 数据映射到 Eigen 的 `SparseMatrix<double, RowMajor>`（利用 `Eigen::Map`，零拷贝）
- **`solve()`** — 使用 Eigen `SimplicialLDLT`（稀疏 Cholesky 分解）求解
- **`calRightVals()`** — 计算 `_rightVals = K_original * u`（用于验算反力）

## 三、MatrixFun 中的函数（`MatrixFun.h/.cpp`）

矩阵向量运算工具库，包含以下函数：

### 基础线性代数运算

| 函数 | 描述 |
|------|------|
| `determinant(matrix)` | 递归计算方阵行列式（代数余子式展开） |
| `inverse(matrix, inv)` | 计算逆矩阵（伴随矩阵法），返回行列式值 |
| `transpose(matrix)` | 矩阵转置 (m×n → n×m) |
| `multiply(A, B)` | 矩阵乘法 (m×n × n×p → m×p) |
| `multiply(A, x)` | 矩阵×向量 (m×n × n → m) |

### 向量运算

| 函数 | 描述 |
|------|------|
| `crossProduct(a, b)` | 三维向量叉积 |
| `dotProduct(a, b)` | 向量点积 |
| `add(a, b)` | 向量加法 |
| `subtract(a, b)` | 向量减法 |
| `scalarMultiply(a, scalar)` | 向量数乘 |
| `norm(a)` | 向量 2-范数（长度） |

### 方向向量计算

| 函数 | 描述 |
|------|------|
| `calcDir2D(r)` | 2D 向量方向：`[cosθ, sinθ, L]`，输入 `(dx, dy)` |
| `calcDir3D(r)` | 3D 向量方向：`[cosα, cosβ, cosγ, L]` |
| `calcDir2D2(r)` | 2D 两点方向：`[cosθ, sinθ, L]`，输入 `(x1,y1, x2,y2)` |
| `calcDir3D2(r)` | 3D 两点方向：`[cosα, cosβ, cosγ, L]`，输入 `(x1,y1,z1, x2,y2,z2)` |

### 坐标变换相关

| 函数 | 描述 |
|------|------|
| `multiplyTAT(T, A)` | 计算 `Tᵀ·A·T`（用于刚度矩阵坐标变换） |
| `multiplyT(T, v)` | 计算 `Tᵀ·v` |
| `computeTransformMatrix(k, n, m, r)` | 计算从全局到局部坐标的变换矩阵 T（Gram-Schmidt 正交化），用于梁单元等需要方向定义的单元 |

## 四、其他文件

### 4.1 CDFEG.h — DLL 导出宏定义

- `CDFEG_API` — Windows 下 `__declspec(dllexport/dllimport)`，控制 DLL 符号导出；Linux 下为空

### 4.2 Processor — 前后处理器基类（`Processor.h/.cpp`）

所有前后处理器的虚基类：
- `_femData` / `_phyFieldData` — 持有有限元数据和物理场数据指针
- `pre()` — 前处理（读网格文件等）
- `post(it)` — 后处理（输出结果文件），`it` 为时间步索引

### 4.3 vtkPost — VTK 后处理器（`vtkPost.h/.cpp`）

继承 Processor，输出 `.vtu` 格式文件，可用 Paraview 可视化。

### 4.4 GidProPost — GiD 前后处理器（新版）（`gidProPost.h/.cpp`）

继承 Processor，读写 GiD 新格式文件（`.dat` 前处理网格 + `.msh`/`.res` 后处理）。

**关键方法**：
- `pre()` — 解析 GiD 格式的网格数据、材料、边界条件
- `readCoord()` / `readElement()` / `readUBF()` — 读取不同数据块
- `gidMsh()` — 生成 GiD 后处理网格文件
- `post()` / `post2()` — 输出 GiD 后处理结果文件

### 4.5 GidPrePost2 — GiD 前后处理器（旧版）（`GidPrePost2.h/.cpp`）

继承 Processor，支持旧版 GiD 文件格式（与 `GidProPost` 不同，兼容老版本）。

### 4.6 GidResItem — GiD 结果项定义（`GidResItem.h/.cpp`）

定义 GiD 后处理结果项的元数据：
- `GidLocation` — 结果位置：`OnNodes`（节点）、`OnGaussPoints`（高斯点）
- `GidResultType` — 结果类型：`Scalar`、`Vector`、`Matrix` 等
- `GidResItem` — 单个结果项，包含名称、类型、场索引、值名称

### 4.7 vtkCellType.h — VTK 单元类型枚举

VTK 标准的单元类型枚举，包括线性单元（VERTEX/TRIANGLE/QUAD/TETRA/HEXAHEDRON 等）、二次单元、高阶单元、Lagrange 单元、Bezier 单元。

### 4.8 InpDataStructures.h — Abaqus INP 数据结构

定义 Abaqus INP 文件解析所用到的所有数据结构：
- `InpNode` / `InpElement` / `InpMaterial` — 节点/单元/材料
- `InpPart` / `InpInstance` — 部件/装配实例（支持 Part-Assembly 结构）
- `InpNodeSet` / `InpElementSet` — 节点集/单元集
- `InpSection` / `BeamSection` / `ConnectorBehavior` — 截面/梁截面/连接器
- `InpStep` / `InpLoad` / `InpBoundary` — 分析步/荷载/边界条件
- `InpAmplitude` — 幅值曲线

### 4.9 inpReader — Abaqus INP 文件解析器（`inpReader.h/.cpp`）

继承 Processor，解析 Abaqus/CAE 导出的 `.inp` 文件：
- 支持 Part-Assembly 结构解析
- 支持多种关键词：`*Node`、`*Element`、`*Material`、`*Elastic`、`*Nset`、`*Elset`、`*Boundary`、`*Part`、`*Instance`、`*Section` 等
- 支持 `*Nset, instance=xxx` 跨实例引用
- 解析结果通过 `getNodes()` / `getElements()` / `getMaterials()` 等访问器获取
- 核心调用 `convertToFEMData()` 将解析结果填入 FEMData 结构

### 4.10 TextReader — 通用文本文件读取器（`TextReader.h/.cpp`）

通用逐行文本解析工具：
- `open()` / `readNextLine()` / `preLine()` — 逐行读取，支持回退
- `simplifyLine()` — 去除注释和多余空格
- `parseInfoLine()` — 解析关键字信息行（如 `*Material, name=Steel`）
- `split()` / `splitDoubles()` / `splitInts()` — 字符串分割和类型转换
- `toLowerCase()` — 大小写不敏感支持
